  • Projects 2

    Projects 2 consisted of a serie of 6 assignments which a future System- and networkadministrator might have to perform. However, to start this course we had to divide ourselves in groups of 4 people and sign-up to our online repository on GitHub.

    The first assignment was a set of network exercises. However, unlike previous exercises the focus was mainly on documenting how you accomplished the tasks.

    This assignment was a sub-set of assignment #2. For this assignment we had to prepare a CentOS 7-server with MariaDB, Firewalld and SELinux installed.

  • Projects 3

    For Projects 3 the class had to divide itself into Team Red or Green. Team Red would develop a network composed entirely of Windows servers. Team Green would do the same, but entirely composed of Linux servers.
    After the teams were divided more or less evenly, we then had to create smaller groups who would focus on one of the following tasks:
    Deploy one of the following servers:

      Active Directory with DNS Fallback AD with DNS Exchange server Mail server DHCP server File server Sharepoint Real-time montering server MSCS Sharepoint

    Or, build the internal and external network connecting all of these devices.
    Because, between these 2 seperate networks we would then have to develop a working connection over VPN.
    Finally, at the end of the assignment, we had to deploy both the network and servers automatically using zero-touch installation software.

  • Bachelorthesis

    My bachelorthesis compared the differences difference between Docker for Windows on a Windows Server 2016 and Docker CE on a CentOS 7.4.
    Specifically, I used software testing to get concrete data on how well each device functions regarding performance and security.
    To accomplish this, I'm using Vagrant to automatically and consistently deploy both servers again and again
    As a conclusion I discovered that DevOps-teams best use Docker for Linux. Because, there is more documentation available for it and the performance is significantly better. However, Microsoft-minded companies and/or users with a Windows computer should make the transistion. Because, the ease of use is quite a bit higher.
